
현재 토큰(token) 방식은 보완

 - 토큰의 만료기간을 최대한 짧게 잡는다. ( 약 10분정도 / xss 공격에 약함! ) -> 토큰을 탈취하는데 최소 3분 걸림!
 - 통상적으로 토큰을 2개로 구분해서 사용함
   > access Token, Refresh Token

(예시)

[ 클라이언트(access Token) 시간 짧게 10분, 해킹당해도 10분뒤면 소용없는거!  ]   [ 서버(Refresh Token) 애는 일주일, 데이터베이스에 있는거면 절대 안뚫림 절대안전? ]
=> 10분지나면 토큰이 사라지면서 자동 로그아웃! (403오류) [ 리프레쉬 토큰이 있다면 10분 더 연장 가능! 사용자입장에서는 로그인을 안해도 계속 연장되고있는거!
